# 🚀 Credential Store Quick Start Guide

Complete PostgreSQL-backed credential management for quick org switching.

---

## 📦 What's Included

✅ **Backend Infrastructure**
- PostgreSQL database with encrypted credential storage
- AES-256-GCM encryption for passwords & security tokens
- RESTful API endpoints for CRUD operations
- Automated setup and testing scripts

✅ **Frontend UI**
- Dropdown selector for saved credentials
- One-click login with saved credentials
- Save credential checkbox on manual login
- Delete credential buttons with confirmation
- Visual credential list with profile names

---

## ⚡ Quick Setup (3 Commands)

```bash
# 1. Run automated setup (creates DB, runs migration, generates encryption key)
bash scripts/setup-credential-store.sh

# 2. Start the development server
npm run dev

# 3. Test the API endpoints (optional)
bash scripts/test-credentials-api.sh
```

**That's it!** Open http://localhost:1717 and you'll see the credential dropdown.

## 🎯 How to Use

### Save a Credential

1. Enter credentials manually (Direct Login tab)
2. Check **"Remember this credential for quick login"**
3. Enter a profile name (e.g., "Production Org")
4. Click "Login with Credentials"
5. ✅ Credential is saved and you're logged in

### Login with Saved Credential

**Option 1: Dropdown**
- Select credential from dropdown at top
- Automatically logs you in

**Option 2: Quick Buttons**
- Click any credential in the list below dropdown
- Automatically logs you in

### Delete a Credential

- Hover over a credential in the list
- Click the trash icon that appears
- Confirm deletion

```bash

# Encryption Key (auto-generated by setup script)

# Optional: Salesforce OAuth (if using custom Connected App)
# SALESFORCE_CLIENT_ID=your_consumer_key

```

### Heroku Production

```bash
# Add PostgreSQL addon (sets DATABASE_URL automatically)
heroku addons:create heroku-postgresql:essential-0

# Set encryption key
heroku config:set CREDENTIAL_ENCRYPTION_KEY=$(openssl rand -hex 32)

# Run migration
heroku pg:psql < migrations/001_create_saved_credentials.sql

# Verify
heroku pg:info
```

## 🐛 Troubleshooting

### Error: "DATABASE_URL environment variable not set"

**Solution:**
```bash
# Check if .env.local exists
cat .env.local | grep DATABASE_URL

# If missing, run setup script again
bash scripts/setup-credential-store.sh
```

### Error: "CREDENTIAL_ENCRYPTION_KEY not set"

**Solution:**
```bash
# Generate new key
openssl rand -hex 32

# Add to .env.local
echo "CREDENTIAL_ENCRYPTION_KEY=<paste_key_here>" >> .env.local
```

### Error: "relation 'saved_credentials' does not exist"

**Solution:**
```bash
# Run migration
psql agentforce_analyzer < migrations/001_create_saved_credentials.sql

# Verify
psql agentforce_analyzer -c "\d saved_credentials"
```

### Error: "Failed to fetch credentials" (in browser console)

**Solution:**
1. Check if PostgreSQL is running: `brew services list` (macOS) or `sudo service postgresql status` (Linux)
2. Check if database exists: `psql -l | grep agentforce`
3. Check server logs for connection errors

### Credentials not appearing in dropdown

**Solution:**
1. Open browser dev tools (F12)
2. Check Network tab for `/api/credentials` request
3. Check Console for errors
4. Verify credentials in database: `psql agentforce_analyzer -c "SELECT * FROM saved_credentials;"`

## 📊 Database Schema

```sql
CREATE TABLE saved_credentials (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  profile_name VARCHAR(255) NOT NULL,
  login_url VARCHAR(500) NOT NULL,
  username VARCHAR(255) NOT NULL,
  password_encrypted TEXT NOT NULL,        -- AES-256-GCM encrypted
  security_token_encrypted TEXT,           -- AES-256-GCM encrypted
  password_iv VARCHAR(32) NOT NULL,        -- Initialization Vector
  token_iv VARCHAR(32),                    -- Initialization Vector
  auth_method VARCHAR(20) NOT NULL CHECK (auth_method IN ('oauth', 'direct')),
  last_used TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  user_id VARCHAR(255),                    -- Future: multi-user support
  CONSTRAINT unique_profile_per_user UNIQUE (profile_name, user_id)
);
```

## 🔐 Security Best Practices

### Development

- ✅ Use strong master password for local PostgreSQL
- ✅ Never commit `.env.local` to Git (already in `.gitignore`)
- ✅ Regenerate encryption key if compromised
- ✅ Use localhost connections only

### Production

- ✅ Enable SSL for PostgreSQL connections
- ✅ Use Heroku Postgres with automatic backups
- ✅ Rotate encryption key periodically
- ✅ Enable Heroku's credential encryption at rest
- ✅ Use HTTPS only (enforced by `secure` cookie flag)
- ✅ Monitor access logs for suspicious activity

### Future Enhancements

- ⏳ Add user authentication (tie credentials to users)
- ⏳ Add 2FA for credential access
- ⏳ Add audit logging for all credential operations
- ⏳ Add credential expiry (auto-delete after 90 days)
- ⏳ Add credential sharing with team members
- ⏳ Add rate limiting to prevent brute force
